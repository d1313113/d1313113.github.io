# 记录一次更新nodejs遇到的问题

# 背景

之前电脑中的nodejs是通过在nodejs官网下载安装包安装的,在后续想更新时使用brew更新,导致双版本存在.

希望将原有版本去除,同时处理好环境变量.

# 安装nodejs

因已经存在了nodejs,又使用了brew更新

brew upgrade node
1

安装完毕,查看版本时

node -v
1

10.x.x 发现依旧是旧版的,不是新版12.x.x

# 处理问题

  1. 首先先删除当前node
brew uninstall node
1
  1. 查看node和npm是否还存在,发现依旧存在
where node
# node not found
where npm
# 连续输出两条
# /usr/local/bin/npm
# /usr/local/bin/npm
1
2
3
4
5
6
  1. 强制删除npm后,重装node
where npm
sudo rm -rf /usr/local/bin/npm
brew install node
1
2
3

此时安装node报错,提示brew link失败 4. 验证node

node -v
1

command not found: node,此时发现node并没有写入环境变量中,但已安装成功

  1. 处理残留ls软连接
brew link --overwrite --dry-run node
brew unintsll node
brew install node
1
2
3
  1. 验证
node -v
# v12.12.0
npm -v
# 6.11.3
1
2
3
4

能正确的输出版本号,此次问题解决.
